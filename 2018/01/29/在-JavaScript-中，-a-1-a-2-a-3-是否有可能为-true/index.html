在 JavaScript 中， (a ==1 && a== 2 && a==3) 是否有可能为 true

发表于 2018-01-29 | 分类于 javaScript

这是最近国外一个求职者在面试时被问到的问题，虽然看到这个问题的时候就想到答案是”yes”,因为在 JavaScript 中，两个”==”是松散相等运算符，所以肯定是有一种方法最后使这个式子为true,但是真的自己写又不会了，就从别处学习了一下别人怎么写：

方法一

自定义 toString（或者 valueOf）方法，每次调用改变一次返回值，从而满足判断条件

注释：
使用“==”时，如果两边类型不同，这个操作符会先转换操作数（强制转型）尝试将其中两边类型转换成相同的然后进行比较，转换不同数据类型，遵循的基本工作：

  1. 如果有一个操作数是布尔值，则在比较相等性之前先将其转换成数值–false转为0，true转为1；
  2. 如果一个操作数是字符串，另一个是数值，在比较相等性之前先将字符串转换成数值；
  3. 如果一个操作数是对象，另一个不是，则调用valueOf()方法，用得到的基本类型值按照前面规则比较
  4. null和undefined是相等的，要比较相等性前，不能将null和undefined转换成其他任何值
  5. 如果其中一个操作数是NaN，则相等操作符返回false

valueOf和toString比较
javascript中所有数据类型都拥有valueOf和toString这两个方法，null除外,toString()函数的作用是用是返回一个反映这个对象的字符串。valueOf()函数的作用是返回它相应的原始值。
